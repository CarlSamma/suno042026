## 📋 What the Wiki Contains

The SUNO15825 project is a **Node.js/Express backend** that transforms `{artist, title?}` inputs into Suno AI–compliant music prompts using a **sequential 5-stage multi-LLM pipeline**. Key findings:

### Architecture at a glance
```
User Input → Express Server → 5-Stage LLM Pipeline → Validator → JSON Response
               ↕                     ↕
           Last.fm API       6 LLM Providers with fallbacks
```

### The 5 Pipeline Stages (in sequence)
| # | Name | Primary LLM | Fallback | Temp | Purpose |
|---|---|---|---|---|---|
| 1 | Analysis | Gemini 1.5 Flash | Mistral | 0.7 | Extract musical DNA |
| 2 | Refinement | Mistral Medium | Google | 0.5 | Organize into 6 categories |
| 3 | Enhancement | DeepSeek Chat | Anthropic | 0.8 | Add creative/evocative language |
| 4 | Quality | Claude 3 Haiku | OpenAI | 0.4 | Polish & verify accuracy |
| 5 | Finalization | GPT-4o Mini | Anthropic | 0.3 | Compress to ≤400 chars |

### Key design decisions documented
- **Fallback logic** — every stage tries its primary, then auto-falls to a backup
- **Prompt Validator** — enforces 400-char limit, no artist names, English-only, checks 4 required element categories
- **Auto-cleanup** — if validation fails, the route automatically runs `cleanPrompt()` + `truncateIntelligently()` before returning
- **Last.fm integration** — non-critical; failures return `null` and never block the pipeline
- **Perplexity** — registered but not used in any stage (future-ready)
- **Cost estimate** ~$4–9.50 per 1000 full pipeline runs
